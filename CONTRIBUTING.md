# Guide de contribution

## Comment contribuer
Le processus est détaillé sur la page `/contribuer` du site web.
1. Forkez le dépôt.
2. Créez une branche (`git checkout -b ma-feature`).
3. Ajoutez le fichier Markdown.
4. Ouvrez la Pull Request.

## Vérifications
Assurez-vous que votre frontmatter correspond bien à l'un des schémas attendus (Étude de cas, Solution TP, Ressource, Implémentation). La CI vérifiera automatiquement ces champs via Zod et rejettera la PR en cas d'erreur de schéma.
